### 🔴 1. AccountAPI 다중 상속 문제 (심각)
**파일**: `pykis/account/api.py:32-33`

```python
class AccountAPI(BaseAPI, ExceptionHandler):
    def __init__(self, ...):
        BaseAPI.__init__(self, client, account_info, enable_cache, cache_config)
        ExceptionHandler.__init__(self, "AccountAPI")
```

**문제점**: `BaseAPI`가 이미 `ExceptionHandler`를 상속받는데 `AccountAPI`가 `ExceptionHandler`를 다시 상속받아 **다이아몬드 상속** 문제 발생. `ExceptionHandler.__init__`이 두 번 호출될 수 있음.

**해결 방안**: `AccountAPI(BaseAPI)`로 단순화하고, `ExceptionHandler.__init__` 직접 호출 제거

### 🟠 2. @api_method 데코레이터 reraise 기본값 불일치
**파일**: `pykis/core/exceptions.py:257`

```python
def api_method(
    context: str,
    reraise: bool = False,  # 기본값: 예외 suppress
    ...
)
```

**문제점**: 
- `@api_method("계좌별 주문 수량 조회")` (reraise=False) → 예외 발생 시 **None 반환** (조용히 실패)
- `@api_method("신용 주문", reraise=True)` → 예외 재발생

**위험**: 주문 관련 메서드가 `reraise=True`이지만 조회 메서드는 `reraise=False`로 **예외가 조용히 무시**됨. 호출자가 None 체크를 놓치면 런타임 에러 발생.

### 🟠 3. 전역 Rate Limiter 설정 변경 무시
**파일**: `pykis/core/rate_limiter.py:31-70`

```python
def get_global_rate_limiter(
    requests_per_second: int = 18,
    ...
) -> "RateLimiter":
    if _global_rate_limiter is None:
        # 첫 호출 시에만 설정 적용
        _global_rate_limiter = RateLimiter(...)
    return _global_rate_limiter  # 이후 호출은 기존 설정 유지
```

**문제점**: 첫 호출 이후 전달된 설정은 무시됨. `Agent`와 `KISClient`에서 서로 다른 설정을 전달해도 먼저 호출된 쪽 설정만 적용.

**예시**:
```python
agent = Agent(rate_limiter_config={"requests_per_second": 5})  # 먼저 생성
client = KISClient()  # requests_per_second=18로 호출하지만 무시됨
```
